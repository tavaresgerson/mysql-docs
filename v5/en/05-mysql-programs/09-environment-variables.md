## 4.9 Variáveis de Ambiente

Esta seção lista variáveis de ambiente que são usadas direta ou indiretamente pelo MySQL. A maioria delas também pode ser encontrada em outras partes deste manual.

Opções na linha de comando têm precedência sobre valores especificados em arquivos de opção e variáveis de ambiente, e valores em arquivos de opção têm precedência sobre valores em variáveis de ambiente. Em muitos casos, é preferível usar um arquivo de opção em vez de variáveis de ambiente para modificar o comportamento do MySQL. Consulte a Seção 4.2.2.2, “Using Option Files”.

Para obter informações sobre o arquivo history do **mysql**, consulte a Seção 4.5.1.3, “mysql Client Logging”.

`MYSQL_TEST_LOGIN_FILE` é o Path do arquivo Login Path (o arquivo criado pelo **mysql_config_editor**). Se não for definido, o valor padrão é o diretório `%APPDATA%\MySQL\.mylogin.cnf` no Windows e `$HOME/.mylogin.cnf` em sistemas que não são Windows. Consulte a Seção 4.6.6, “mysql_config_editor — MySQL Configuration Utility”.

As variáveis `MYSQL_TEST_TRACE_DEBUG` e `MYSQL_TEST_TRACE_CRASH` controlam o Client Plugin de Trace do protocolo de teste, se o MySQL for compilado com esse Plugin habilitado. Para mais informações, consulte Using the Test Protocol Trace Plugin.

Os valores padrão de `UMASK` e `UMASK_DIR` são `0640` e `0750`, respectivamente. O MySQL assume que o valor para `UMASK` ou `UMASK_DIR` está em octal se começar com zero. Por exemplo, definir `UMASK=0600` é equivalente a `UMASK=384` porque 0600 octal é 384 decimal.

As variáveis `UMASK` e `UMASK_DIR`, apesar de seus nomes, são usadas como modos, não como masks (máscaras):

* Se `UMASK` for definido, o **mysqld** usa `($UMASK | 0600)` como o modo para a criação de arquivos, de modo que os arquivos recém-criados tenham um modo no intervalo de 0600 a 0666 (todos os valores em octal).

* Se `UMASK_DIR` for definido, o **mysqld** usa `($UMASK_DIR | 0700)` como o modo base para a criação de diretórios, que é então aplicado em operação AND com `~(~$UMASK & 0666)`, de modo que os diretórios recém-criados tenham um modo no intervalo de 0700 a 0777 (todos os valores em octal). A operação AND pode remover permissões de leitura e escrita (read and write) do modo do diretório, mas não permissões de execução (execute).

Consulte também a Seção B.3.3.1, “Problems with File Permissions”.

Pode ser necessário definir `PKG_CONFIG_PATH` se você usar o **pkg-config** para compilar programas MySQL. Consulte Building C API Client Programs Using pkg-config.
