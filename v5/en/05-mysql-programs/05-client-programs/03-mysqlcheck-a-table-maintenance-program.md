### 4.5.3 mysqlcheck — Um Programa de Manutenção de Tabela

O `client` **mysqlcheck** executa a manutenção de tabela: Ele verifica, repara, otimiza ou analisa tabelas.

Cada tabela é travada (`locked`) e, portanto, indisponível para outras sessões enquanto está sendo processada, embora para operações de `check`, a tabela seja travada apenas com um `READ lock` (consulte a Seção 13.3.5, “LOCK TABLES and UNLOCK TABLES Statements”, para mais informações sobre `READ` e `WRITE locks`). As operações de manutenção de tabela podem consumir muito tempo, particularmente para tabelas grandes. Se você usar a opção `--databases` ou `--all-databases` para processar todas as tabelas em um ou mais Databases, uma invocação do **mysqlcheck** pode demorar muito. (Isso também se aplica ao procedimento de `upgrade` do MySQL se ele determinar que a verificação de tabelas é necessária, pois ele processa as tabelas da mesma maneira.)

O **mysqlcheck** deve ser usado enquanto o `mysqld` server estiver em execução, o que significa que você não precisa parar o server para realizar a manutenção da tabela.

O **mysqlcheck** usa as instruções `SQL` `CHECK TABLE`, `REPAIR TABLE`, `ANALYZE TABLE` e `OPTIMIZE TABLE` de uma maneira conveniente para o usuário. Ele determina quais instruções usar para a operação que você deseja executar e, em seguida, envia as instruções para o server para serem executadas. Para detalhes sobre com quais `storage engines` cada instrução funciona, veja as descrições dessas instruções na Seção 13.7.2, “Table Maintenance Statements”.

Nem todos os `storage engines` suportam necessariamente as quatro operações de manutenção. Nesses casos, uma mensagem de erro é exibida. Por exemplo, se `test.t` for uma tabela `MEMORY`, uma tentativa de verificá-la produz este resultado:

```sql
$> mysqlcheck test t
test.t
note     : The storage engine for the table doesn't support check
```

Se o **mysqlcheck** não conseguir reparar uma tabela, consulte a Seção 2.10.12, “Rebuilding or Repairing Tables or Indexes” para estratégias manuais de reparo de tabela. Este é o caso, por exemplo, para tabelas `InnoDB`, que podem ser verificadas com `CHECK TABLE`, mas não reparadas com `REPAIR TABLE`.

Atenção

É melhor fazer um backup de uma tabela antes de executar uma operação de reparo de tabela; em algumas circunstâncias, a operação pode causar perda de dados. Causas possíveis incluem, mas não se limitam a, erros no sistema de arquivos.

Existem três formas gerais de invocar o **mysqlcheck**:

```sql
mysqlcheck [options] db_name [tbl_name ...]
mysqlcheck [options] --databases db_name ...
mysqlcheck [options] --all-databases
```

Se você não nomear nenhuma tabela após *`db_name`* ou se usar a opção `--databases` ou `--all-databases`, Databases inteiros são verificados.

O **mysqlcheck** tem um recurso especial em comparação com outros programas `client`. O comportamento padrão de verificar tabelas (`--check`) pode ser alterado renomeando o binário. Se você quiser ter uma ferramenta que repare tabelas por padrão, você deve apenas fazer uma cópia do **mysqlcheck** chamada **mysqlrepair**, ou criar um link simbólico para **mysqlcheck** chamado **mysqlrepair**. Se você invocar **mysqlrepair**, ele repara as tabelas.
