#### 5.4.4.4 Formato de Log para Alterações nas Tabelas do Database mysql

O conteúdo das tabelas de concessão (grant tables) no database `mysql` pode ser modificado direta ou indiretamente. Diretamente (por exemplo, com [`INSERT`](insert.html "13.2.5 INSERT Statement") ou [`DELETE`](delete.html "13.2.2 DELETE Statement")) ou indiretamente (por exemplo, com [`GRANT`](grant.html "13.7.1.4 GRANT Statement") ou [`CREATE USER`](create-user.html "13.7.1.2 CREATE USER Statement")). Statements que afetam as tabelas do database `mysql` são escritas no binary log usando as seguintes regras:

* Statements de manipulação de dados que alteram diretamente dados nas tabelas do database `mysql` são logadas de acordo com a configuração da variável de sistema [`binlog_format`](replication-options-binary-log.html#sysvar_binlog_format). Isso se aplica a statements como [`INSERT`](insert.html "13.2.5 INSERT Statement"), [`UPDATE`](update.html "13.2.11 UPDATE Statement"), [`DELETE`](delete.html "13.2.2 DELETE Statement"), [`REPLACE`](replace.html "13.2.8 REPLACE Statement"), [`DO`](do.html "13.2.3 DO Statement"), [`LOAD DATA`](load-data.html "13.2.6 LOAD DATA Statement"), [`SELECT`](select.html "13.2.9 SELECT Statement") e [`TRUNCATE TABLE`](truncate-table.html "13.1.34 TRUNCATE TABLE Statement").

* Statements que alteram o database `mysql` indiretamente são logadas como statements, independentemente do valor de [`binlog_format`](replication-options-binary-log.html#sysvar_binlog_format). Isso se aplica a statements como [`GRANT`](grant.html "13.7.1.4 GRANT Statement"), [`REVOKE`](revoke.html "13.7.1.6 REVOKE Statement"), [`SET PASSWORD`](set-password.html "13.7.1.7 SET PASSWORD Statement"), [`RENAME USER`](rename-user.html "13.7.1.5 RENAME USER Statement"), `CREATE` (todas as formas, exceto [`CREATE TABLE ... SELECT`](create-table.html "13.1.18 CREATE TABLE Statement")), `ALTER` (todas as formas) e `DROP` (todas as formas).

[`CREATE TABLE ... SELECT`](create-table.html "13.1.18 CREATE TABLE Statement") é uma combinação de definição e manipulação de dados. A parte [`CREATE TABLE`](create-table.html "13.1.18 CREATE TABLE Statement") é logada usando o formato statement e a parte [`SELECT`](select.html "13.2.9 SELECT Statement") é logada de acordo com o valor de [`binlog_format`](replication-options-binary-log.html#sysvar_binlog_format).
