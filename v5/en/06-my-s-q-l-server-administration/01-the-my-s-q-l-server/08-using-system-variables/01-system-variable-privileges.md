#### 5.1.8.1 Privilégios de System Variable

Uma system variable pode ter um valor global que afeta a operação do server como um todo, um valor de session que afeta apenas a session atual, ou ambos. Para modificar valores de runtime de system variable, use a instrução [`SET`](set-variable.html "13.7.4.1 SET Syntax for Variable Assignment"). Consulte a [Seção 13.7.4.1, “SET Syntax for Variable Assignment”](set-variable.html "13.7.4.1 SET Syntax for Variable Assignment"). Esta seção descreve os privilégios necessários para atribuir valores a system variables em runtime.

Definir um valor de runtime para uma system variable global requer o privilégio [`SUPER`](privileges-provided.html#priv_super).

Para definir um valor de runtime de uma system variable de session, use a instrução [`SET SESSION`](set-variable.html "13.7.4.1 SET Syntax for Variable Assignment"). Em contraste com a definição de valores de runtime globais, definir valores de runtime de session normalmente não requer privilégios especiais e pode ser feito por qualquer usuário para afetar a session atual. Para algumas system variables, definir o valor de session pode ter efeitos fora da session atual e, portanto, é uma operação restrita que só pode ser realizada por usuários que possuam o privilégio [`SUPER`](privileges-provided.html#priv_super). Se uma system variable de session for restrita dessa forma, a descrição da variável indicará essa restrição. Exemplos incluem [`binlog_format`](replication-options-binary-log.html#sysvar_binlog_format) e [`sql_log_bin`](replication-options-binary-log.html#sysvar_sql_log_bin). Definir o valor de session dessas variáveis afeta o binary logging para a session atual, mas também pode ter implicações mais amplas para a integridade da replication e dos backups do server.
