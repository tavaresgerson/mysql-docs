#### 5.8.1.1 Compilando o MySQL para Debugging

Se você tiver algum problema muito específico, pode sempre tentar fazer o debug do MySQL. Para isso, você deve configurar o MySQL com a opção [`-DWITH_DEBUG=1`](source-configuration-options.html#option_cmake_with_debug). Você pode verificar se o MySQL foi compilado com debugging executando: [**mysqld --help**](mysqld.html "4.3.1 mysqld — The MySQL Server"). Se o flag [`--debug`](server-options.html#option_mysqld_debug) estiver listado com as opções, então o debugging está habilitado. [**mysqladmin ver**](mysqladmin.html "4.5.2 mysqladmin — A MySQL Server Administration Program") também lista a versão do [**mysqld**](mysqld.html "4.3.1 mysqld — The MySQL Server") como [**mysql ... --debug**](mysql.html "4.5.1 mysql — The MySQL Command-Line Client") neste caso.

Se o [**mysqld**](mysqld.html "4.3.1 mysqld — The MySQL Server") parar de falhar (crashing) quando você o configura com a opção CMake [`-DWITH_DEBUG=1`](source-configuration-options.html#option_cmake_with_debug), você provavelmente encontrou um bug no compiler ou um timing bug dentro do MySQL. Neste caso, você pode tentar adicionar `-g` usando as opções CMake [`CMAKE_C_FLAGS`](source-configuration-options.html#option_cmake_cmake_c_flags) e [`CMAKE_CXX_FLAGS`](source-configuration-options.html#option_cmake_cmake_cxx_flags) e não usar [`-DWITH_DEBUG=1`](source-configuration-options.html#option_cmake_with_debug). Se o [**mysqld**](mysqld.html "4.3.1 mysqld — The MySQL Server") morrer, você pode pelo menos anexá-lo com o **gdb** ou usar o **gdb** no core file para descobrir o que aconteceu.

Quando você configura o MySQL para debugging, você habilita automaticamente muitas funções extras de verificação de segurança (safety check) que monitoram a integridade do [**mysqld**](mysqld.html "4.3.1 mysqld — The MySQL Server"). Se elas encontrarem algo “inesperado”, uma entrada é escrita para o `stderr`, que o [**mysqld_safe**](mysqld-safe.html "4.3.2 mysqld_safe — MySQL Server Startup Script") direciona para o error log! Isso também significa que se você estiver tendo problemas inesperados com o MySQL e estiver usando uma source distribution, a primeira coisa que você deve fazer é configurar o MySQL para debugging. Se você acredita ter encontrado um bug, por favor, utilize as instruções na [Seção 1.5, “Como Reportar Bugs ou Problemas”](bug-reports.html "1.5 How to Report Bugs or Problems").

Na distribuição MySQL para Windows, o `mysqld.exe` é compilado por padrão com suporte a trace files.
