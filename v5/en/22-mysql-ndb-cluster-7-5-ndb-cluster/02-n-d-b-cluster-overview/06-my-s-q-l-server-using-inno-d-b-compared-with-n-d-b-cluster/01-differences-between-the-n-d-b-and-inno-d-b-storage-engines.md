#### 21.2.6.1 Differences Between the NDB and InnoDB Storage Engines

The [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") storage engine is implemented using a distributed, shared-nothing architecture, which causes it to behave differently from [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine") in a number of ways. For those unaccustomed to working with [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6"), unexpected behaviors can arise due to its distributed nature with regard to transactions, foreign keys, table limits, and other characteristics. These are shown in the following table:
