#### B.3.3.7 Problemas de Time Zone

Se você tiver um problema com `SELECT NOW()` retornando valores em UTC e não em sua hora local, você deve informar ao server o seu `time zone` atual. O mesmo se aplica se [`UNIX_TIMESTAMP()`](date-and-time-functions.html#function_unix-timestamp) retornar o valor incorreto. Isso deve ser configurado no environment em que o server é executado (por exemplo, em [**mysqld_safe**](mysqld-safe.html "4.3.2 mysqld_safe — MySQL Server Startup Script") ou [**mysql.server**](mysql-server.html "4.3.3 mysql.server — MySQL Server Startup Script")). Consulte [Seção 4.9, “Environment Variables”](environment-variables.html "4.9 Environment Variables").

Você pode definir o `time zone` para o server usando a opção [`--timezone=timezone_name`](mysqld-safe.html#option_mysqld_safe_timezone) no [**mysqld_safe**](mysqld-safe.html "4.3.2 mysqld_safe — MySQL Server Startup Script"). Você também pode configurá-lo definindo a environment variable `TZ` antes de iniciar o [**mysqld**](mysqld.html "4.3.1 mysqld — The MySQL Server").

Os valores permitidos para [`--timezone`](mysqld-safe.html#option_mysqld_safe_timezone) ou `TZ` são dependentes do sistema. Consulte a documentação do seu sistema operacional para verificar quais valores são aceitáveis.
