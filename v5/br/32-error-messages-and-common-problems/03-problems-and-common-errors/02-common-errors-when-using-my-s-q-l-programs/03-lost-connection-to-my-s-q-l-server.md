#### B.3.2.3 Lost connection to MySQL server

Existem três causas prováveis para esta mensagem de erro.

Geralmente, isso indica problemas de conectividade de rede e você deve verificar a condição da sua rede se este erro ocorrer frequentemente. Se a mensagem de erro incluir “during query” (durante a Query), este é provavelmente o caso que você está enfrentando.

Às vezes, o formato “during query” ocorre quando milhões de linhas estão sendo enviadas como parte de uma ou mais Queries. Se você souber que isso está acontecendo, você deve tentar aumentar o [`net_read_timeout`](server-system-variables.html#sysvar_net_read_timeout) de seu valor padrão de 30 segundos para 60 segundos ou mais, o suficiente para que a transferência de dados seja concluída.

Mais raramente, pode acontecer quando o client está tentando a conexão inicial com o server. Neste caso, se o valor do seu [`connect_timeout`](server-system-variables.html#sysvar_connect_timeout) estiver definido para apenas alguns segundos, você pode resolver o problema aumentando-o para dez segundos, talvez mais, se você tiver uma conexão de longa distância ou lenta. Você pode determinar se está enfrentando esta causa incomum usando `SHOW GLOBAL STATUS LIKE 'Aborted_connects'`. Este valor aumenta em um para cada tentativa de conexão inicial que o server aborta. Você pode ver “reading authorization packet” como parte da mensagem de erro; se for o caso, isso também sugere que esta é a solução de que você precisa.

Se a causa não for nenhuma das descritas acima, você pode estar enfrentando um problema com valores [`BLOB`](blob.html "11.3.4 The BLOB and TEXT Types") que são maiores do que o [`max_allowed_packet`](server-system-variables.html#sysvar_max_allowed_packet), o que pode causar este erro em alguns clients. Às vezes, você pode ver um erro [`ER_NET_PACKET_TOO_LARGE`](/doc/mysql-errors/5.7/en/server-error-reference.html#error_er_net_packet_too_large), e isso confirma que você precisa aumentar o [`max_allowed_packet`](server-system-variables.html#sysvar_max_allowed_packet).
