### 8.2.5 Otimizando privilégios do banco de dados

Quanto mais complexo for a configuração do privilégio, mais overhead é aplicado a todas as instruções SQL. Simplificar os privilégios estabelecidos pelas instruções `GRANT` permite que o MySQL reduza o overhead de verificação de permissões quando os clientes executam instruções. Por exemplo, se você não conceder quaisquer privilégios de nível de tabela ou coluna, o servidor não precisará verificar o conteúdo das tabelas `tables_priv` e `columns_priv`. Da mesma forma, se você não definir limites de recursos para nenhuma conta, o servidor não precisa realizar o contagem de recursos. Se você tiver uma carga muito alta de processamento de instruções, considere usar uma estrutura de concessão simplificada para reduzir o overhead de verificação de permissões.
