## 28.8 Visão Geral do MySQL Enterprise Monitor

Para informações sobre o comportamento do MySQL Enterprise Monitor, consulte o manual do MySQL Enterprise Monitor: https://dev.mysql.com/doc/mysql-monitor/en/.

Importante

O MySQL Enterprise Monitor chegará ao fim de vida (End of Life - EOL) e será descontinuado por obsolescência a partir de 1º de janeiro de 2025.

Após essa data, o MySQL Enterprise Monitor não receberá mais Security Updates, Non-Security Updates, Bug Fixes ou atualizações de conteúdo técnico online. Ele fará a transição para o modelo *Sustaining Support*.

O que esperar quando o MySQL Enterprise Monitor atingir o fim de vida (EOL):

* O MySQL encerrará todas as atividades de Bug Fix para o produto
* O MySQL encerrará todas as atividades de Security Fix para o produto
* O MySQL encerrará todo o trabalho de desenvolvimento de novos recursos para o produto

O Sustaining Support não inclui:

* Novas atualizações de programa, fixes, Security Alerts e Critical Patch Updates

* Novas atualizações fiscais, legais ou regulatórias
* Novos scripts de upgrade
* Certificação com novos produtos/versões de terceiros
* Comprometimento de 24 horas e diretrizes de resposta para Severity 1 service requests, conforme definido na "Seção 9 - Definições de Severidade" no documento intitulado "Oracle Software Technical Support Policies"

* Fixes ou atualizações lançadas anteriormente que a Oracle tenha retirado de publicação. Pacotes de software publicados mais antigos ou existentes permanecerão disponíveis como conteúdo arquivado.

Para o conjunto de Oracle Technical Support Policies, visite: <https://www.oracle.com/support/policies.html>

Para uma explicação dos diferentes modelos de suporte (como Sustaining Support), visite: <https://www.oracle.com/support/lifetime-support/>

Para clientes que estão usando versões anteriores do MySQL Enterprise Monitor atualmente, suas opções incluem:

* Usar o Enterprise Manager for MySQL. Este é um produto gratuito para clientes com um Oracle Support Contract válido. Para mais informações, visite [Comprehensive Monitoring and Compliance Management for MySQL Databases using Enterprise Manager](https://blogs.oracle.com/observability/post/comprehensive-monitoring-and-compliance-management-for-mysql-databases-using-enterprise-manager).

* Usar os recursos de Database Monitoring do serviço OCI Database Management e do serviço Ops Insights. Isso está disponível para HeatWave MySQL e MySQL on-premises. Para mais informações, visite [Database Management for MySQL HeatWave and on-premises MySQL](https://blogs.oracle.com/mysql/post/database-management-for-mysql-heatwave) e [Ops Insights for HeatWave and on-premises MySQL](https://docs.oracle.com/en-us/iaas/operations-insights/doc/mysql-insights.html).
