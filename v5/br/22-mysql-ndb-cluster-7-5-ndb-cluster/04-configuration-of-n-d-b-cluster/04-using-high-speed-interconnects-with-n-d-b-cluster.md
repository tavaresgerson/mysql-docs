### 21.4.4 Usando Interconexões de Alta Velocidade com NDB Cluster

Mesmo antes de o projeto do [`NDBCLUSTER`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") começar em 1996, era evidente que um dos principais problemas a serem encontrados na construção de Databases paralelas seria a comunicação entre os Nodes na rede. Por esta razão, o [`NDBCLUSTER`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") foi projetado desde o início para permitir o uso de diversos mecanismos de transporte de dados diferentes, ou *transporters*.

NDB Cluster 7.5 e 7.6 suportam três destes (veja [Seção 21.2.1, “NDB Cluster Core Concepts”](mysql-cluster-basics.html "21.2.1 NDB Cluster Core Concepts")). Um quarto *transporter*, Scalable Coherent Interface (SCI), também era suportado em versões muito antigas do `NDB`. Isso exigia hardware, software e binários MySQL especializados que não estão mais disponíveis.
