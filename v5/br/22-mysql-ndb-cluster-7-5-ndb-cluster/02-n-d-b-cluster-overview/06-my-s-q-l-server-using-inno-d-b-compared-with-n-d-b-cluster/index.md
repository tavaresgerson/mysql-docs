### 21.2.6 MySQL Server Usando InnoDB Comparado com NDB Cluster

[21.2.6.1 Diferenças entre as Storage Engines NDB e InnoDB](mysql-cluster-ndb-innodb-engines.html)

[21.2.6.2 Workloads de NDB e InnoDB](mysql-cluster-ndb-innodb-workloads.html)

[21.2.6.3 Resumo de Uso de Recursos do NDB e InnoDB](mysql-cluster-ndb-innodb-usage.html)

O MySQL Server oferece várias opções de storage engines. Como tanto o [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") quanto o [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine") podem servir como storage engines transacionais do MySQL, os usuários do MySQL Server às vezes se interessam pelo NDB Cluster. Eles veem o [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") como uma possível alternativa ou upgrade para a storage engine [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine") padrão no MySQL 5.7. Embora [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") e [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine") compartilhem características comuns, existem diferenças na arquitetura e na implementação, o que significa que alguns aplicativos existentes do MySQL Server e cenários de uso podem ser adequados para o NDB Cluster, mas não todos eles.

Nesta seção, discutimos e comparamos algumas características da storage engine [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") usada pelo NDB 7.5 com o [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine") usado no MySQL 5.7. As próximas seções fornecem uma comparação técnica. Em muitos casos, as decisões sobre quando e onde usar o NDB Cluster devem ser tomadas caso a caso, considerando todos os fatores. Embora esteja além do escopo desta documentação fornecer detalhes para cada cenário de uso concebível, também tentamos oferecer algumas orientações muito gerais sobre a adequação relativa de alguns tipos comuns de aplicações para back ends [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") em oposição aos back ends [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine").

O NDB Cluster 7.5 usa um [**mysqld**](mysqld.html "4.3.1 mysqld — The MySQL Server") baseado no MySQL 5.7, incluindo suporte para [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine") 1.1. Embora seja possível usar tabelas `InnoDB` com o NDB Cluster, essas tabelas não são clustered. Também não é possível usar programas ou bibliotecas de uma distribuição NDB Cluster 7.5 com o MySQL Server 5.7, ou o inverso.

Embora também seja verdade que alguns tipos de aplicações de negócios comuns podem ser executadas tanto no NDB Cluster quanto no MySQL Server (muito provavelmente usando a storage engine [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine")), existem algumas diferenças arquitetônicas e de implementação importantes. [Seção 21.2.6.1, “Diferenças entre as Storage Engines NDB e InnoDB”](mysql-cluster-ndb-innodb-engines.html "21.2.6.1 Differences Between the NDB and InnoDB Storage Engines"), fornece um resumo dessas diferenças. Devido a essas diferenças, alguns cenários de uso são claramente mais adequados para uma engine ou para a outra; veja [Seção 21.2.6.2, “Workloads de NDB e InnoDB”](mysql-cluster-ndb-innodb-workloads.html "21.2.6.2 NDB and InnoDB Workloads"). Isso, por sua vez, impacta os tipos de aplicações mais adequadas para uso com [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") ou [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine"). Consulte [Seção 21.2.6.3, “Resumo de Uso de Recursos do NDB e InnoDB”](mysql-cluster-ndb-innodb-usage.html "21.2.6.3 NDB and InnoDB Feature Usage Summary"), para uma comparação da adequação relativa de cada um para uso em tipos comuns de aplicações de Database.
