#### 21.2.6.2 Cargas de Trabalho (Workloads) NDB e InnoDB

O NDB Cluster possui uma série de atributos exclusivos que o tornam ideal para atender aplicações que exigem alta disponibilidade, failover rápido, alto throughput e baixa latência. Devido à sua arquitetura distribuída e implementação multi-node, o NDB Cluster também possui restrições específicas que podem impedir que algumas cargas de trabalho (workloads) apresentem bom desempenho. Uma série de grandes diferenças de comportamento entre os Storage Engines [`NDB`](mysql-cluster.html "Chapter 21 MySQL NDB Cluster 7.5 and NDB Cluster 7.6") e [`InnoDB`](innodb-storage-engine.html "Chapter 14 The InnoDB Storage Engine") em relação a alguns tipos comuns de cargas de trabalho de aplicações orientadas a Database são mostradas na tabela a seguir:
