#### 21.6.11.2 Uso de Links Simbólicos com Objetos de Dados de Disco

O desempenho de um NDB Cluster que utiliza o armazenamento de dados em disco pode ser significativamente melhorado ao separar o sistema de arquivos do nó de dados de quaisquer arquivos de espaço de tabelas (arquivos de log de desfazer e arquivos de dados) e colocá-los em discos diferentes. Nas versões iniciais do NDB Cluster, não havia suporte direto para essa separação no NDB Cluster, e era necessário realizar essa separação usando links simbólicos. O NDB Cluster agora suporta os parâmetros de configuração do nó de dados `FileSystemPathDD`, `FileSystemPathDataFiles` e `FileSystemPathUndoFiles`, o que torna o uso de links simbólicos para esse propósito desnecessário. Para obter mais informações sobre esses parâmetros, consulte Parâmetros do sistema de arquivos de dados em disco.
