### 16.1.1 Visão Geral da Configuração da Replicação Baseada na Posição do Arquivo Binary Log

Esta seção descreve a replicação entre servidores MySQL baseada no método de posição do arquivo Binary Log, onde a instância MySQL operando como a Source (onde as alterações do Database se originam) escreve atualizações e alterações como "eventos" no Binary Log. As informações no Binary Log são armazenadas em diferentes formatos de logging, dependendo das alterações do Database que estão sendo registradas. As Replicas são configuradas para ler o Binary Log da Source e executar os eventos no Binary Log no Database local da Replica.

Cada Replica recebe uma cópia do conteúdo completo do Binary Log. É responsabilidade da Replica decidir quais statements no Binary Log devem ser executados. A menos que você especifique o contrário, todos os eventos no Binary Log da Source são executados na Replica. Se necessário, você pode configurar a Replica para processar apenas eventos que se aplicam a Databases ou tabelas específicas.

Importante

Você não pode configurar a Source para logar apenas certos eventos.

Cada Replica mantém um registro das coordenadas do Binary Log: o nome do arquivo e a position dentro do arquivo que foi lida e processada a partir da Source. Isso significa que múltiplas Replicas podem estar conectadas à Source e executando partes diferentes do mesmo Binary Log. Como as Replicas controlam esse processo, Replicas individuais podem ser conectadas e desconectadas do servidor sem afetar a operação da Source. Além disso, como cada Replica registra a position atual dentro do Binary Log, é possível que as Replicas sejam desconectadas, reconectem e, em seguida, retomem o processamento.

A Source e cada Replica devem ser configuradas com um ID exclusivo (usando a variável de sistema [`server_id`](replication-options.html#sysvar_server_id)). Além disso, cada Replica deve ser configurada com informações sobre o nome do host da Source, o nome do arquivo de log e a position dentro desse arquivo. Esses detalhes podem ser controlados a partir de uma sessão MySQL usando a instrução [`CHANGE MASTER TO`](change-master-to.html "13.4.2.1 CHANGE MASTER TO Statement") na Replica. Os detalhes são armazenados no repositório de metadata de conexão da Replica, que pode ser um arquivo ou uma tabela (consulte [Section 16.2.4, “Relay Log and Replication Metadata Repositories”](replica-logs.html "16.2.4 Relay Log and Replication Metadata Repositories")).
