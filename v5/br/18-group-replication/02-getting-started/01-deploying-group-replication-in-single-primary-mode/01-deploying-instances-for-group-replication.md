#### 17.2.1.1 Deploying Instances for Group Replication

O primeiro passo é realizar o deploy de pelo menos três instances do MySQL Server. Este procedimento demonstra o uso de múltiplos hosts para as instances, nomeados s1, s2 e s3. Presume-se que o MySQL Server esteja instalado em cada host (veja [Capítulo 2, *Instalando e Atualizando o MySQL*](installing.html "Capítulo 2 Instalando e Atualizando o MySQL")). O plugin Group Replication é fornecido com o MySQL Server 5.7.17 e posterior; nenhum software adicional é necessário, embora o plugin deva ser instalado no MySQL Server em execução. Veja [Seção 17.2.1.1, “Deploying Instances for Group Replication”](group-replication-deploying-instances.html "17.2.1.1 Deploying Instances for Group Replication"); para informações adicionais, veja [Seção 5.5, “MySQL Server Plugins”](server-plugins.html "5.5 MySQL Server Plugins").

Neste exemplo, três instances são usadas para o group, que é o número mínimo de instances para criar um group. Adicionar mais instances aumenta a fault tolerance do group. Por exemplo, se o group consistir de três members, em caso de failure de uma instance, o group pode continuar. Mas em caso de outro failure, o group não pode mais continuar processando write transactions. Ao adicionar mais instances, o número de servers que podem falhar enquanto o group continua a processar transactions também aumenta. O número máximo de instances que podem ser usadas em um group é nove. Para mais informações, veja [Seção 17.1.3.2, “Failure Detection”](group-replication-failure-detection.html "17.1.3.2 Failure Detection").
