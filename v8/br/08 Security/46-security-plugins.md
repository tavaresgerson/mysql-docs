## 8.4 Componentes e plugins de segurança

O MySQL inclui vários componentes e plugins que implementam recursos de segurança:

* Plugins para autenticar as tentativas de conexão dos clientes com o servidor MySQL. Os plugins estão disponíveis para vários protocolos de autenticação. Para uma discussão geral sobre o processo de autenticação, consulte a Seção 8.2.17, “Plugins de Autenticação”. Para características de plugins de autenticação específicos, consulte a Seção 8.4.1, “Plugins de Autenticação”.
* Um componente de validação de senha para implementar políticas de força de senha e avaliar a força de senhas potenciais. Consulte a Seção 8.4.3, “O Componente de Validação de Senhas”.
* Plugins de cartela de chaves que fornecem armazenamento seguro para informações sensíveis. Consulte a Seção 8.4.4, “A Cartela de Chaves MySQL”.
* (Apenas a Edição Empresarial do MySQL) O MySQL Enterprise Audit, implementado usando um plugin do servidor, utiliza a API de auditoria MySQL aberta para habilitar monitoramento e registro padrão e baseado em políticas da atividade de conexão e consulta executada em servidores MySQL específicos. Projetado para atender à especificação de auditoria da Oracle, o MySQL Enterprise Audit fornece uma solução de auditoria e conformidade fácil de usar, pronta para uso, para aplicativos que são regidos por diretrizes regulatórias internas e externas. Consulte a Seção 8.4.5, “MySQL Enterprise Audit”.
* Uma função permite que os aplicativos adicionem seus próprios eventos de mensagem ao log de auditoria. Consulte a Seção 8.4.6, “O Componente de Mensagem de Auditoria”.
* (Apenas a Edição Empresarial do MySQL) O MySQL Enterprise Firewall, um firewall de nível de aplicativo que permite que os administradores de banco de dados permitam ou negam a execução de instruções SQL com base na correspondência com listas de padrões de instruções aceitos. Isso ajuda a proteger o MySQL Server contra ataques como injeção SQL ou tentativas de explorar aplicativos usando-os fora de suas características legítimas de carga de consulta. Consulte a Seção 8.4.7, “MySQL Enterprise Firewall”.
* (Apenas a Edição Empresarial do MySQL) O MySQL Enterprise Data Masking and De-Identification, implementado como uma biblioteca de plugins contendo um plugin e um conjunto de funções. O mascaramento de dados oculta informações sensíveis substituindo valores reais por substitutos. As funções de mascaramento de dados do MySQL Enterprise Data Masking and De-Identification permitem mascarar dados existentes usando vários métodos, como ofuscação (removendo características identificadoras), geração de dados aleatórios formatados e substituição ou substituição de dados. Consulte a Seção 8.5, “MySQL Enterprise Data Masking and De-Identification”.
