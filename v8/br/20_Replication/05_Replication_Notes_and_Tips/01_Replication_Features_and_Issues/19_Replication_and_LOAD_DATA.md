#### 19.5.1.19 Replicação e LOAD DATA

`LOAD DATA` é considerado inseguro para o registro baseado em declarações (veja a Seção 19.2.1.3, “Determinação de Declarações Seguras e Inseguras no Registro Binário”). Quando `binlog_format=MIXED` é definido, a declaração é registrada no formato baseado em linhas. Quando `binlog_format=STATEMENT` é definido, observe que `LOAD DATA` não gera uma mensagem de aviso, ao contrário de outras declarações inseguras.

Se você usar `LOAD DATA` com `binlog_format=STATEMENT`, cada replica na qual as alterações devem ser aplicadas cria um arquivo temporário contendo os dados. A replica então usa uma instrução `LOAD DATA` para aplicar as alterações. Esse arquivo temporário não é criptografado, mesmo que a criptografia do log binário esteja ativa na fonte. Se a criptografia for necessária, use o formato de registro binário baseado em linhas ou misto, para o qual as réplicas não criam o arquivo temporário.

Se uma conta `PRIVILEGE_CHECKS_USER` tiver sido usada para ajudar a garantir o canal de replicação (consulte a Seção 19.3.3, “Verificação de privilégios de replicação”), é altamente recomendável que você registre as operações `LOAD DATA` usando o registro binário baseado em linhas (`binlog_format=ROW`). Se o canal estiver configurado para `REQUIRE_ROW_FORMAT`, o registro binário baseado em linhas é obrigatório. Com esse formato de registro, o privilégio `FILE` não é necessário para executar o evento, então não dê esse privilégio à conta `PRIVILEGE_CHECKS_USER`. Se você precisar recuperar de um erro de replicação envolvendo uma operação `LOAD DATA INFILE` registrada no formato de declaração, e o evento replicado é confiável, você pode conceder o privilégio `FILE` à conta `PRIVILEGE_CHECKS_USER` temporariamente, removendo-o após o evento replicado ter sido aplicado.

Quando o **mysqlbinlog** lê eventos de log para instruções `LOAD DATA` registradas no formato baseado em instruções, um arquivo local gerado é criado em um diretório temporário. Esses arquivos temporários não são removidos automaticamente pelo **mysqlbinlog** ou por qualquer outro programa do MySQL. Se você usar instruções `LOAD DATA` com registro binário baseado em instruções, você deve excluir os arquivos temporários você mesmo depois de não precisar mais do log da instrução. Para mais informações, consulte a Seção 6.6.9, “mysqlbinlog — Ferramenta para Processamento de Arquivos de Log Binário”.
