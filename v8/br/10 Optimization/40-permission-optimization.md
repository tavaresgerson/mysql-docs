### 10.2.6 Otimizando os Privilegios do Banco de Dados

Quanto mais complexo for a configuração dos privilégios, maior será o overhead aplicado a todas as instruções SQL. Simplificar os privilégios estabelecidos pelas instruções `GRANT` permite que o MySQL reduza o overhead de verificação de permissões quando os clientes executam instruções. Por exemplo, se você não conceder quaisquer privilégios de nível de tabela ou coluna, o servidor não precisará verificar o conteúdo das tabelas `tables_priv` e `columns_priv`. Da mesma forma, se você não definir limites de recursos para nenhuma conta, o servidor não precisará realizar a contagem de recursos. Se você tiver uma carga de processamento de instruções muito alta, considere usar uma estrutura de concessão simplificada para reduzir o overhead de verificação de permissões.
