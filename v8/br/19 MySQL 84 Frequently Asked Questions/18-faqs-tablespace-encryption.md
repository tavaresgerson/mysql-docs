## A.17 Perguntas frequentes sobre o MySQL 8.4: Criptografia de dados em repouso do InnoDB

<table><colgroup><col align="left" width="1%"/><col/></colgroup><tbody><tr class="question"><td align="left" valign="top"><p><b>A.17.1.</b></p></td><td align="left" valign="top"><p> <b>A.17.1.</b> Dados são descriptografados para usuários autorizados a visualizá-los? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> <b>Sim.</b> A criptografia de dados em repouso do <code>InnoDB</code> é projetada para aplicar a criptografia de forma transparente dentro do banco de dados, sem impactar aplicações existentes. A devolução de dados em formato criptografado quebraria a maioria das aplicações existentes. A criptografia de dados em repouso do <code>InnoDB</code> oferece o benefício da criptografia sem o overhead associado às soluções tradicionais de criptografia de bancos de dados, que normalmente exigiriam mudanças caras e substanciais em aplicações, gatilhos de banco de dados e visualizações. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.17.2.</b></p></td><td align="left" valign="top"><p> Qual é o overhead associado à criptografia de dados em repouso do <code>InnoDB</code>? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> Não há overhead de armazenamento adicional. De acordo com benchmarks internos, o overhead de desempenho equivale a uma diferença de porcentagem de dígitos únicos. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.17.3.</b></p></td><td align="left" valign="top"><p> Quais algoritmos de criptografia são usados com a criptografia de dados em repouso do <code>InnoDB</code>? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> A criptografia de dados em repouso do <code>InnoDB</code> suporta o algoritmo de criptografia avançada (AES256) de bloco. Utiliza o modo de criptografia de bloco Electronic Codebook (ECB) para a criptografia de chave do espaço de tabelas e o modo de criptografia de bloco Cipher Block Chaining (CBC) para a criptografia de dados. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.17.4.</b></p></td><td align="left" valign="top"><p> É possível usar algoritmos de criptografia de terceiros no lugar do algoritmo fornecido pela funcionalidade de criptografia de dados em repouso do <code>InnoDB</code>? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> Não, não é possível usar outros algoritmos de criptografia. O algoritmo de criptografia fornecido é amplamente aceito. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.17.5.</b></p></td><td align="left" valign="top"><p> Colunas indexadas podem ser criptografadas? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> A criptografia de dados em repouso do <code>InnoDB</code> suporta todos os índices de forma transparente. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.17.6.</b></p></td><td align="left" valign="top"><p> Quais tipos de dados e comprimentos de dados são suportados pela criptografia de dados em repouso do <code>InnoDB</code>? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> A criptografia de dados em repouso do <code>InnoDB</code> suporta todos os tipos de dados suportados. Não há limitação de comprimento de dados. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.17.7.</b></p></td><td align="left" valign="top"><p> Os dados permanecem criptografados na rede? </p></td></tr><
