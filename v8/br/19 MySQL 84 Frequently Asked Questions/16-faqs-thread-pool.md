## A.15 Perguntas frequentes sobre o MySQL 8.4: Pool de threads do MySQL Enterprise

<table><colgroup><col align="left" width="1%"/><col/></colgroup><tbody><tr class="question"><td align="left" valign="top"><p><b>A.15.1.</b></p></td><td align="left" valign="top"><p><b>A.15.1.1.</b> O que é o Pool de Sêries e que problema ele resolve? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> O Pool de Sêries do MySQL é um plugin do servidor MySQL que estende as capacidades padrão de manipulação de conexões do servidor MySQL para limitar o número de instruções/consultas e transações executadas simultaneamente para garantir que cada uma tenha recursos de CPU e memória suficientes para cumprir sua tarefa. Para o MySQL 8.4, o plugin de Pool de Sêries está incluído na Edição Empresarial do MySQL, um produto comercial. </p><p> O modelo padrão de manipulação de threads no MySQL Server executa instruções usando um thread por conexão de cliente. À medida que mais clientes se conectam ao servidor e executam instruções, o desempenho geral degrada. O plugin de Pool de Sêries oferece um modelo alternativo de manipulação de threads projetado para reduzir o overhead e melhorar o desempenho. O plugin de Pool de Sêries aumenta o desempenho do servidor gerenciando eficientemente os threads de execução de instruções para um grande número de conexões de clientes, especialmente em sistemas modernos com múltiplos núcleos/cores. </p><p> Para mais informações, consulte a Seção 7.6.3, “Pool de Sêries do MySQL Enterprise”. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.15.2.</b></p></td><td align="left" valign="top"><p> Como o Pool de Sêries limita e gerencia sessões e transações concorrentes para um desempenho e throughput ótimos? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> O Pool de Sêries usa uma abordagem de <span class="quote">“<span class="quote">dividir e conquistar</span>”</span> para limitar e equilibrar a concorrência. Ao contrário da manipulação padrão de conexões do MySQL Server, o Pool de Sêries separa conexões e threads, então não há uma relação fixa entre conexões e os threads que executam instruções recebidas dessas conexões. O Pool de Sêries, então, gerencia conexões de clientes dentro de grupos de threads configuráveis, onde são priorizadas e colocadas em fila com base na natureza do trabalho que foram submetidas a serem realizadas. </p><p> Para mais informações, consulte a Seção 7.6.3.3, “Operação do Pool de Sêries”. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.15.3.</b></p></td><td align="left" valign="top"><p> Como o Pool de Sêries difere do Pool de Conexões no lado do cliente? </p></td></tr><tr class="answer"><td align="left" valign="top"></td><td align="left" valign="top"><p> O Pool de Conexões do MySQL opera no lado do cliente para garantir que um cliente MySQL não se conecte e desconecte constantemente do servidor MySQL. Ele é projetado para armazenar conexões ociosas no cliente MySQL para uso por outros usuários conforme necessário. Isso minimiza o overhead e o custo de estabelecer e desfazer conexões à medida que as consultas são enviadas ao servidor MySQL. O Pool de Conexões do MySQL não tem visibilidade sobre as capacidades de manipulação de consultas ou carga do servidor MySQL backend. Em contraste, o Pool de Sêries opera no lado do servidor MySQL e é projetado para gerenciar a execução de conexões e consultas concorrentes recebidas conforme elas são recebidas das conexões de clientes acessando o banco de dados MySQL backend. Devido à separação de funções, o Pool de Conexões do MySQL e o Pool de Sêries são ortogonais e podem ser usados independentemente um do outro. </p><p> O Pool de Conexões do MySQL é coberto no Capítulo 31, <i>Conectores e APIs</i>. </p></td></tr><tr class="question"><td align="left" valign="top"><p><b>A.15.4.</b></p></td><td align="left" valign="top"><p> Quando devo usar o Pool de Sêries
