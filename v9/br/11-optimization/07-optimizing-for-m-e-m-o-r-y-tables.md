## 10.7 Otimizando para Tabelas de MEMÓRIA

Considere usar tabelas `MEMORY` para dados não críticos que são acessados frequentemente e são de leitura somente ou raramente atualizados. Compare sua aplicação com tabelas `InnoDB` ou `MyISAM` equivalentes sob uma carga de trabalho realista, para confirmar se o desempenho adicional vale o risco de perder dados ou o overhead de copiar dados de uma tabela baseada em disco no início da aplicação.

Para obter o melhor desempenho com tabelas `MEMORY`, examine os tipos de consultas em cada tabela e especifique o tipo a ser usado para cada índice associado, seja um índice B-tree ou um índice hash. Na instrução `CREATE INDEX`, use a cláusula `USING BTREE` ou `USING HASH`. Os índices B-tree são rápidos para consultas que realizam comparações maior que ou menor que por meio de operadores como `>` ou `BETWEEN`. Os índices hash são rápidos apenas para consultas que buscam valores únicos por meio do operador `=` ou um conjunto restrito de valores por meio do operador `IN`. Para saber por que `USING BTREE` é frequentemente uma escolha melhor do que o padrão `USING HASH`, consulte a Seção 10.2.1.23, “Evitar varreduras completas da tabela”. Para detalhes de implementação dos diferentes tipos de índices `MEMORY`, consulte a Seção 10.3.9, “Comparação de índices B-tree e hash”.
