#### 20.8.1.1 Versões dos membros durante as atualizações

Durante um procedimento de atualização online, se o grupo estiver no modo de único primário, todos os servidores que não estão atualmente fora de linha para a atualização funcionam da mesma forma que antes. O grupo elege um novo primário sempre que necessário, seguindo as políticas de eleição descritas na Seção 20.1.3.1, “Modo de único primário”. Observe que, se você precisar que o primário permaneça o mesmo durante toda a atualização (exceto quando ele está sendo atualizado), você deve primeiro atualizar todos os secundários para uma versão superior ou igual à versão do membro primário alvo, e depois atualizar o primário por último. O primário não pode permanecer como primário a menos que esteja executando a versão mais baixa do MySQL Server no grupo. Após a atualização do primário, você pode usar a função `group_replication_set_as_primary()` para reapotá-lo como primário.

Se o grupo estiver no modo de múltiplos primários, menos membros online estarão disponíveis para realizar escritas durante o procedimento de atualização, porque os membros atualizados se juntam no modo de leitura apenas após sua atualização. Quando todos os membros tiverem sido atualizados para a mesma versão, eles todos voltam automaticamente ao modo de leitura/escrita.

Para lidar com uma situação de problema, por exemplo, se você precisar reverter uma atualização para uma versão anterior da versão principal ou adicionar capacidade extra a um grupo em uma emergência, é possível permitir que um membro se junte a um grupo online a partir de uma versão que, de outra forma, não poderia.
