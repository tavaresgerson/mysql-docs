#### 25.2.6.2 Cargas de trabalho NDB e InnoDB

O NDB Cluster possui uma série de atributos únicos que o tornam ideal para atender aplicações que exigem alta disponibilidade, failover rápido, alto desempenho e baixa latência. Devido à sua arquitetura distribuída e implementação em vários nós, o NDB Cluster também tem restrições específicas que podem impedir o bom desempenho de algumas cargas de trabalho. Várias diferenças significativas no comportamento entre os motores de armazenamento `NDB` e `InnoDB` em relação a alguns tipos comuns de cargas de trabalho de aplicativos baseados em banco de dados são mostradas na tabela a seguir:
